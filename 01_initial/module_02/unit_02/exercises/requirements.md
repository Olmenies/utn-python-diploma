# Ejercicios Unidad 02
## Introducción
### Propósito:
El propósito de esta unidad es implementar la Persistencia de Datos, asegurando que la información de tu proyecto no se pierda al cerrar el programa. Para esto, se utilizarán archivos y bases de datos relacionales.

- Migrar a la Persistencia: Evolucionar de las listas temporales a sistemas de almacenamiento permanente.
- Dominar SQLite3: Aprender a crear tablas y realizar operaciones de base de datos directamente desde Python.
- Gestionar Archivos: Utilizar formatos de texto (.txt) y serialización para guardar configuraciones o regisros.

### ¿Por qué te conviene resolver esta guía?
Sin persistencia, tu aplicación "olvida" todoo cada vez que se apaga. Al resolver estos ejericicios, estarás contruyendo la base de datos de tu trabajo final, permitiendo que los registros de tus producos o clientes queden guardados para siempre.

### Importante:
Estas tareas pueden intentar hacerlas hasta la fecha límite nidicada, en donde vamos a hacerlas juntos paso a paso. Es conveniente que aún cuando no las puedan realizar solos, lo intentes.

Consulten todas las dudas en el foro y anoten las mismas para conversar durante la clase online.
## Consignas
### Tarea 01 - Almacenamiento en Estructuras:
A partir del ejercicio de la unidad anterior, cree una aplicación que permita realizar un **alta** de registros guardandolos dentro de un **dicionario** interno de la app.

### Tarea 02 - Conexión a Base de Datos - Hito del Proyecto:
A partir del ejercicio de la unidad anterior, evolucione la aplicación para que el **alta** de registros se realice de forma definitiva en una base de datos **SQLite3**.

### Tarea 03 - Registro en Archivos Planos:
A partir del ejercicio de la unidad antrerior, cree una versión de la aplicación que permita realizar el **alta** de los registros capturados en un archivo de texto interno (.txt).
